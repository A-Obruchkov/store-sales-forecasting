# Прогнозирование продаж (Time Series Forecasting)

## Описание проекта

Проект посвящен прогнозированию продаж в розничной сети на основе исторических данных.  
Рассматриваются различные уровни агрегации данных и сравниваются подходы к моделированию.

В рамках проекта решаются задачи:
- прогнозирование продаж для одной категории в одном магазине
- прогнозирование для всех категорий в одном магазине
- прогнозирование для всех категорий во всех магазинах

## Бизнес-цель

Основная цель — повысить качество планирования:
- оптимизация закупок
- снижение излишков и списаний
- повышение доступности товаров
- улучшение операционной эффективности

## Данные

Используются исторические данные продаж:
- период: с 2013-01-01 по 2016-03
- уровень: магазины и товарные категории
- дополнительные источники:
  - цены на нефть
  - календарь праздников

## Структура проекта

В репозитории представлены три основных ноутбука:

### 1. one-family-one-store-forecast.ipynb
Прогноз продаж для одной категории (BREAD/BAKERY) в одном магазине.

Содержит:
- разведочный анализ данных (EDA)
- анализ сезонности и трендов
- feature engineering
- базовый (наивный) прогноз

### 2. all-family-one-stores-forecast.ipynb
Прогноз для всех категорий в одном магазине.

Содержит:
- подготовку данных
- проверку гипотез
- сравнение моделей машинного обучения:
  - LightGBM
  - CatBoost
  - XGBoost
- оценку качества по метрике RMSLE

### 3. all-family-all-stores-forecast.ipynb
Прогноз для всех категорий во всех магазинах.

Содержит:
- масштабирование подхода
- сравнение гипотез
- анализ влияния данных и признаков
- интерпретацию результатов

## Подход

В проекте сравниваются несколько подходов:

1. Наивный прогноз  
   Используется как baseline (например, значение продаж 16 дней назад)

2. Модели машинного обучения  
   Используются алгоритмы:
   - XGBoost
   - LightGBM
   - CatBoost

3. Анализ данных  
   - влияние сезонности
   - влияние старых данных
   - значимость признаков

## Метрика качества

Для оценки используется метрика:

**RMSLE (Root Mean Squared Logarithmic Error)**

Она хорошо подходит для задач прогнозирования продаж, так как:
- снижает влияние выбросов
- учитывает относительные ошибки

## Основные результаты

- Модели машинного обучения значительно превосходят наивный прогноз
- XGBoost показывает наилучшее качество среди протестированных моделей
- Исторические данные (включая ранние периоды) оказывают существенное влияние на качество прогноза
- Учет сезонности и календарных факторов улучшает результат
